Build a complete, production-ready, full-stack web-based Accounting & Business Management application. The app must be deployable to any cloud server (Railway, Render, AWS, GCP, etc.).

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
TECH STACK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Backend: Node.js + Express.js (REST API)
- Database: PostgreSQL with Prisma ORM (schema migrations included)
- Frontend: React.js + Tailwind CSS (single-page app, responsive)
- Auth: JWT-based login with role-based access (Admin, Manager, Staff, Driver)
- PDF generation: Puppeteer or pdfkit (for bills, reports)
- Thermal print: ESC/POS-compatible HTML template (80mm)
- File export: xlsx and CSV export for reports
- Deployment: Dockerfile + docker-compose.yml included
- Environment: .env file with all config (DB URL, JWT secret, port, company info)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MODULE 1 — SALES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Sale Invoice
   - Auto invoice number (prefix + date + sequence)
   - Party name (from party master), billing address, GST number
   - Line items: item, HSN/SAC, qty, unit, rate, discount %, GST %,  taxable amount, CGST, SGST, IGST, total
   - Supports both GST and non-GST mode
   - Multiple payment modes on a single invoice: UPI, Cash, Cheque, Bank Transfer, Credit
   - Balance due = total - amount paid
   - Bill format options: A4 and 80mm thermal (toggle before print)
   - Print button (opens print preview)
   - Print with acknowledgement copy (duplicate slip at bottom)
   - Share bill via WhatsApp link (generate shareable HTML/PDF link)

2. Order Booking (phone orders)
   - Create an order with party details, items, delivery address, notes
   - Status: Pending → Confirmed → Dispatched → Delivered
   - Convert order to Sale Invoice in one click
   - After conversion, go directly to Payment page

3. Walk-in Sale
   - Quick sale bill creation (no order step)
   - Directly opens Payment page after saving
   - Payment page supports: Cash, UPI (show QR), Cheque, Multiple modes split

4. Delivery Assignment
   - After creating any sale bill, assign it to one or more drivers
   - Select pickup van/vehicle from vehicle master
   - Multiple bills can be assigned to one driver in one trip
   - Driver assignment screen shows: driver name, vehicle, list of bills, total amount, delivery status
   - Print a delivery manifest (list of all bills for a driver trip)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MODULE 2 — PURCHASE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Purchase Invoice: supplier, date, invoice no, items with HSN, qty, rate, GST
- Purchase Order: create PO → receive goods → auto-update stock on receive
- Debit Note: raise debit note against a purchase (returns or adjustments)
- Credit Note: issue credit note against a sale (returns)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MODULE 3 — ACCOUNTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Journal Entry: date, narration, multiple debit/credit ledger lines, auto-balance check
- Payment Voucher: pay to party/vendor, select ledger, payment mode
- Receipt Voucher: receive from party, select ledger, payment mode
- Contra Entry: cash-to-bank or bank-to-cash transfers

Party Master:
  - Name, address, GSTIN, PAN, phone, email, credit limit, opening balance
  - Type: Customer / Supplier / Both

Ledger Master:
  - Name, group (Assets, Liabilities, Income, Expense, Capital)
  - Opening balance, nature (Dr/Cr)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MODULE 4 — REPORTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
All reports must have: date range filter, search, export to PDF and Excel.

1. Balance Sheet — assets vs liabilities as of any date
2. Profit & Loss Statement — income vs expense for any period, gross profit, net profit
3. Day Book — all vouchers of a day chronologically
4. Sale Register — all sale invoices, party-wise or date-wise, with GST breakup
5. Purchase Register — all purchase invoices with GST breakup
6. Party-wise Ledger:
   - Full transaction history for any party
   - Opening balance → all Dr/Cr entries → closing balance
   - "Share with Party" button: generates a PDF/link and opens WhatsApp with party's number pre-filled
7. Cash Book & Bank Book
8. Trial Balance

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MODULE 5 — GST
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- GSTR-3B Report: summary of outward/inward supplies, ITC, net tax payable
- GSTR-2B Report: ITC available from purchase invoices
- HSN Summary Report: HSN-wise taxable value and GST
- GST Export: export all reports in JSON format (for offline GSTN upload) and Excel
- Reverse Charge toggle on purchase vouchers
- Composition scheme toggle in settings

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MODULE 6 — INVENTORY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Stock Category Master:
  - Category → Sub-category → Brand hierarchy

Stock Item Master:
  - Name, category, brand, HSN code, unit (pcs/kg/ltr/box), purchase rate, sale rate
  - Min stock level (low stock alert)
  - Barcode/SKU field

Stock Transactions:
  - Auto-deduct on sale invoice creation
  - Auto-add on purchase invoice creation
  - Manual stock adjustment entry

Stock Reports:
  - Current Stock: shows Physical Stock and Available Stock separately
    * Physical Stock = actual stock on hand
    * Available Stock = Physical Stock minus stock reserved for pending/confirmed orders
  - Stock Ledger: item-wise in/out history
  - Low Stock Alert report
  - Sale-wise stock movement
  - Purchase-wise stock movement
